# 영훈이형 대화 기반 기능 요구사항

기록일: 2026-06-14  
출처: 카카오톡 대화 (영훈이형 — 부동산 업계 종사자)

---

## 배경

부동산 관련 앱을 만들 때 어떤 기능이 있으면 좋은지 물어본 결과.  
"네이버부동산에 없는 기능 위주로"라는 핵심 방향 제시.

---

## 요청된 핵심 기능 목록

### 1. 거래/가격 정보
- **실거래가** — 신고된 실제 체결가 (국토부 공공데이터포털)
- **평수** — 공급/전용 면적 함께 표시

### 2. 토지/건물 정보
- **토지구획분류** — 용도지역 (1종주거/2종주거/준주거 등)
- **재개발·재건축 대상구역 여부** — 구역 지정 여부 태그
- **건축연한** — 준공연도 및 경과 연수
- **관리비** — 월 관리비 정보

### 3. 입지 분석
- **역세권** — 지하철역까지 도보 거리 (가까운 역명, 도보 분)
- **학세권** — 학교 근처 여부
  - 영훈이형 코멘트: "하닉(학닉?) 상승 서세권 핫하대"
  - 서울 상급학군 인근이 현재 핫한 키워드
- **학군 분석**
  - 각 고등학교별 **인서울 합격률** 데이터 연동
  - 단순 근처 학교 여부가 아니라 **학군 품질** 수치화
- **대기업 셔틀버스 정류장 근접 여부**
  - 대상: SK하이닉스, 삼성전자 셔틀버스 정류장
  - 직주근접 수요 파악 — 셔틀 탑승 가능 위치면 임차/매수 수요 높음
  - 데이터 소스: 각 사 셔틀 노선 공개 정보 또는 수동 수집 필요
  - 매물 좌표 기준 가장 가까운 정류장명 + 도보 거리 표시

### 4. 지역 범위
- **자양2동** 등 세부 행정동 단위 필터
- 전국 단위 데이터 수집 염두 (프로토타입 이후 확장)

---

## 개발 방향 (대화에서 언급된 힌트)

| 항목 | 내용 |
|------|------|
| 단계 | 프로토타입 먼저, 이후 영훈이형에게 다시 연락 |
| 범위 | 일단 자양동 → 이후 전국 확장 가능 구조 |
| 차별화 | 네이버부동산에 없는 기능 (학군 품질, 재개발 태그 등) |
| 기타 | "기타동동많지" — 추가 아이디어 있음, 나중에 재확인 필요 |

---

## 구현 우선순위 (초안)

1. **실거래가 조회** — 국토부 API (키 발급 후 즉시 구현 가능)
2. **역세권 계산** — 지하철역 좌표 데이터 + 매물 좌표 거리 계산
3. **재개발·재건축 구역 태그** — 서울시 공공데이터 활용 가능
4. **토지용도 분류** — 토지이음(eum.go.kr) 또는 국토부 API
5. **학군 분석** — 고교별 인서울 합격률 (수집 방법 별도 검토 필요)
6. **셔틀버스 근접 여부** — SK하이닉스·삼성전자 정류장 좌표 수집 후 거리 계산

---

## 미확인 / 추후 확인 필요

- "올해보너스나오운" — 문맥 불명확, 다른 얘기일 가능성
- "하닉 상승 서새권" — 서울 학군 핵심 키워드 정확한 의미 재확인
- 건축연한, 관리비 데이터 소스 (공공 API 존재 여부 확인 필요)
- 학군 인서울 합격률 데이터 출처 (공식 공개 데이터 vs 스크래핑)
- SK하이닉스·삼성전자 셔틀 정류장 좌표 — 공식 공개 여부 확인 필요, 없으면 수동 수집
